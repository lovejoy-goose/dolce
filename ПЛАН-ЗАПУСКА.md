## 3. Хостинг
**Ответственный:** — sillygoose
**Статус:** ⬜

**Вариант A (старт, бесплатно): GitHub Pages** — сейчас сайт собран под него (папка `docs/`, файл `CNAME`).
- Плюсы: бесплатно, быстро.
- Риски:
  - периодические блокировки/замедления GitHub в РФ → часть локальных клиентов не откроет без VPN;
  - сложно/нельзя привязать домен через A-запись на российский IP;
  - Яндекс хуже ранжирует локальный бизнес на зарубежном хостинге (серверная геолокация — региональный сигнал).
- Вывод: «пожить» можно как временное решение. **Если остаёмся на Pages — обязательно вручную задать регион «Петрозаводск» в Яндекс.Вебмастере: это перебивает зарубежную геолокацию сервера.**

**Вариант B (рекомендуется): российский хостинг**
- reg.ru (~100–150 ₽/мес) или Timeweb — для статики с запасом.
- Домен и хостинг в одном месте — проще привязка.
- Нет рисков блокировки, лучше для Яндекса.

Чек-лист:
- [ ] Выбрать вариант (A или B)

- [ ] Привязать домен
- [ ] Подключить HTTPS (Let's Encrypt — бесплатно)
- [ ] Проверить, что сайт открывается по https и без www → с www (редирект)
