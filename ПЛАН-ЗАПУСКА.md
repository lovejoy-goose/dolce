## 3. Хостинг
**Ответственный:** — sillygoose
**Статус:** ⬜

**Вариант A (старт, бесплатно): GitHub Pages** — сейчас сайт собран под него (папка `docs/`, файл `CNAME`).
- Плюсы: бесплатно, быстро.
- Риски:
  - периодические блокировки/замедления GitHub в РФ → часть локальных клиентов не откроет без VPN;
  - сложно/нельзя привязать домен через A-запись на российский IP;
  - Яндекс хуже ранжирует локальный бизнес на зарубежном хостинге (серверная геолокация — региональный сигнал).
- Вывод: «пожить» можно как временное решение. **Если остаёмся на Pages — обязательно вручную задать регион «Петрозаводск» в Яндекс.Вебмастере: это перебивает зарубежную геолокацию сервера.**

**Вариант B (рекомендуется): российский хостинг**
- reg.ru (~100–150 ₽/мес) или Timeweb — для статики с запасом.
- Домен и хостинг в одном месте — проще привязка.
- Нет рисков блокировки, лучше для Яндекса.

Чек-лист:
- [ ] Выбрать вариант (A или B)
- [ ] Развернуть хостинг

- [ ] Подключить HTTPS (Let's Encrypt — бесплатно)

## 6. Аналитика: Яндекс.Метрика + Вебвизор
**Ответственный:** — sillygoose
**Статус:** ⬜

> Метрика показывает, сколько людей заходит, откуда, по каким запросам, что нажимают.
> **Вебвизор** — запись действий посетителя (как видео): видно, куда смотрят, где уходят. Очень полезно для одностраничника.

**Что нужно:**
- Аккаунт Яндекса (тот же, что для Вебмастера — удобно связать).

**Шаги:**
- [ ] Зайти в [metrika.yandex.ru](https://metrika.yandex.ru) → «Добавить счётчик»
- [ ] Указать имя и адрес сайта (домен)
- [ ] **Включить галочку «Вебвизор, карта скроллинга, аналитика форм»** при создании
- [ ] Вставить **только номер счётчика** в `index.html` → функция `loadMetrika`, строка `COUNTER_ID = null`. Обвязка (загрузка + согласие + Вебвизор) уже готова, весь сниппет вставлять не нужно.
- [ ] В Вебмастере связать сайт со счётчиком Метрики (ускоряет и улучшает данные)
- [ ] Проверить в Метрике, что данные пошли (вкладка «Сводка» через несколько минут)

**Важно (юридически):** счётчик Метрики ставит cookie и собирает данные о посетителе → после его установки нужны **cookie-уведомление и политика обработки ПД** (см. п. 7).
